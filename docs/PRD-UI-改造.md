## 一、产品目标

### 1.1 背景
当前股票回测系统采用浅色主题，视觉风格偏向通用工具，缺乏专业金融终端的沉浸感和信息密度。用户反馈希望改造为类似**同花顺/东方财富专业版**的深色金融终端风格，以提升长期使用舒适度和专业感。

### 1.2 产品目标
- **视觉升级**：从浅色渐变主题切换为深色专业金融终端风格，降低长时间盯盘的视觉疲劳
- **信息密度提升**：参考专业交易软件布局，在有限屏幕空间内展示更多有效信息
- **回测能力增强**：将回测系统从"演示级"提升为"可用级"，支持更完整的策略参数配置和结果分析
- **多端适配**：一套代码同时支持桌面端和移动端访问
- **零成本改造**：全部使用免费CDN资源，不引入付费库或服务

### 1.3 成功指标
- 用户使用时长提升（目标 +30%）
- 回测功能使用率提升（目标 +50%）
- 移动端访问占比（目标 >30%）

## 二、用户故事

### 2.1 核心用户画像
| 角色 | 描述 | 核心需求 |
|------|------|---------|
| 个人投资者 | 有股票投资经验，偶尔做策略回测 | 快速查看K线、简单回测、移动端可用 |
| 量化爱好者 | 熟悉技术指标，希望验证策略想法 | 多种指标、灵活的参数设置、详细回测报告 |
| 产品经理（本人） | 评估系统能力 | 功能完整性、UI专业度 |

### 2.2 用户故事列表

**作为一名个人投资者，我希望：**
- 打开系统就能看到深色主题的K线图，晚上使用不刺眼
- 像使用同花顺一样，左侧有导航栏可以快速切换功能
- 在手机上也能查看K线和回测结果

**作为一名量化爱好者，我希望：**
- 回测系统支持自定义策略参数（均线周期、止盈止损等）
- 回测结果有详细的绩效指标（夏普比率、最大回撤、胜率等）
- 能同时对比多个策略的表现
- K线图支持多种技术指标叠加显示

**作为一名产品决策者，我希望：**
- UI风格统一、专业，达到商业软件水准
- 系统稳定，无明显Bug
- 后续维护成本低，技术栈不复杂

| P0-4 | 回测系统基础功能保留并增强 | 双均线/MACD策略可正常运行，参数可配置 |
| P0-5 | 响应式布局基础适配 | 桌面端(>1024px)和移动端(<768px)均能正常使用 |
| P0-6 | 股票搜索功能深色适配 | 搜索框、下拉候选列表深色风格 |

| 编号 | 需求描述 | 验收标准 |

## 四、UI设计规范

### 4.1 颜色规范

参考同花顺/东方财富专业版深色主题，建立统一色彩体系：

#### 主色板
```
背景色层级：
- 主背景：     #0D1117 (最接近黑的蓝灰，主体背景)
- 二级背景：   #161B22 (卡片/面板背景)
- 三级背景：   #21262D (悬浮/弹出层背景)
- 分割线：     #30363D (边框/分割线)

文字色：
- 主文字：     #E6EDF3 (正文)
- 次级文字：   #8B949E (辅助信息)
- 禁用文字：   #484F58 (禁用状态)

功能色（A股惯例：红涨绿跌）：
- 上涨/买入：  #FF4D4D (红色，A股上涨)
- 下跌/卖出：  #00C853 (绿色，A股下跌)
- 平盘：       #8B949E (灰色)
- 主强调色：   #58A6FF (蓝色，链接/按钮)
- 警告色：     #F0883E (橙色，警告)
- 成功色：     #3FB950 (绿色，成功提示)
```

#### 图表配色（Lightweight Charts）
```javascript
// K线配色
upColor: '#FF4D4D'        // 上涨蜡烛红色
downColor: '#00C853'       // 下跌蜡烛绿色
borderUpColor: '#FF4D4D'
borderDownColor: '#00C853'

// 背景
chartBackground: '#0D1117'
gridColor: '#21262D'
crosshairColor: '#58A6FF'

// MA线配色
MA5:  '#F0883E'  // 橙色
MA10: '#58A6FF'  // 蓝色
MA20: '#3FB950'  // 绿色
MA60: '#FF69B4'  // 粉色
```

### 4.2 字体规范

```
中文字体栈：
font-family: "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;

英文字体栈（数字/代码）：
font-family: "JetBrains Mono", "Fira Code", "Consolas", monospace;

字号规范：
- 页面标题：   20px / font-weight: 600
- 模块标题：   16px / font-weight: 600
- 正文/数据： 14px / font-weight: 400
- 辅助文字：  12px / font-weight: 400
- 超小提示：  11px / font-weight: 400

数字对齐：
所有数字使用等宽字体，确保小数点对齐
```

### 4.3 间距规范

```
基础单位：4px

内边距：
- 卡片内边距：   16px (4 * 4px)
- 按钮内边距：   8px 16px
- 输入框内边距： 8px 12px
- 紧凑模式：     8px

外边距：
- 模块间距：     24px
- 卡片间距：     16px
- 元素间距：     8px

圆角：
- 卡片圆角：     8px
- 按钮圆角：     6px
- 输入框圆角：   6px
- 圆形头像/图标：50%
```

### 4.4 组件规范

#### 按钮
```
主要按钮：背景 #58A6FF，文字白色，hover亮度+10%
次要按钮：背景透明，边框 #58A6FF，文字 #58A6FF
危险按钮：背景 #FF4D4D，文字白色
文字按钮：无背景，文字 #58A6FF，hover下划线
```

#### 输入框/下拉框
```
背景：   #161B22
边框：   #30363D (默认) / #58A6FF (聚焦)
文字：   #E6EDF3
占位符： #484F58
```

#### 表格/数据列表
```
表头背景： #21262D
表头文字： #8B949E，12px，大写
行背景：   交替 #0D1117 / #161B22
行hover：  #21262D
```

## 七、技术实现要点

### 7.1 深色主题实现方案

使用CSS变量统一管理颜色，便于后续维护和新手理解：

```css
:root {
  /* 背景色 */
  --bg-primary: #0D1117;
  --bg-secondary: #161B22;
  --bg-tertiary: #21262D;
  --border-color: #30363D;

  /* 文字色 */
  --text-primary: #E6EDF3;
  --text-secondary: #8B949E;
  --text-disabled: #484F58;

  /* 功能色 */
  --color-rise: #FF4D4D;    /* A股红涨 */
  --color-fall: #00C853;    /* A股绿跌 */
  --color-primary: #58A6FF;
  --color-warning: #F0883E;
  --color-success: #3FB950;
}
```

### 7.2 Lightweight Charts 深色配置

```javascript
const chart = createChart(container, {
  layout: {
    background: { type: 'solid', color: '#0D1117' },
    textColor: '#E6EDF3',
    fontSize: 12,
    fontFamily: '"JetBrains Mono", monospace',
  },
  grid: {
    vertLines: { color: '#21262D' },
    horzLines: { color: '#21262D' },
  },
  crosshair: {
    vertLine: { color: '#58A6FF', width: 1 },
    horzLine: { color: '#58A6FF', width: 1 },
  },
});
```


## 八、待确认问题

1. **数据来源**：当前使用模拟数据还是接入真实API？如果是真实API，是否需要处理跨域问题？
   - 建议：保留模拟数据作为Demo模式，同时预留真实API接口

2. **回测引擎复杂度**：是否需要在前端实现完整的回测引擎，还是调用后端服务？
   - 当前方案：纯前端JavaScript回测引擎（保持零成本）
   - 限制：数据量受浏览器内存限制，建议单次回测不超过2年日线数据

3. **实时行情更新频率**：当前"实时"是轮询还是WebSocket？
   - 建议：使用定时轮询（免费API通常不支持WebSocket）

4. **移动端K线交互**：手机上如何操作K线图的技术指标切换？
   - 建议：底部Tab栏 + 手势操作（双指缩放、单指拖动）

5. **CDN稳定性**：全部使用CDN是否存在版本漂移风险？
   - 建议：锁定CDN版本号，不使用 `latest` 标签

6. **A股/美股/港股**：当前系统支持哪些市场？UI改造是否需要考虑多市场切换？
   - 建议：v1先专注A股，后续扩展
