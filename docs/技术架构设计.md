```
关键特性：
- ✅ 纯前端实现，无需后端服务器
- ✅ 支持 file:// 协议直接打开（内嵌数据方案）
- ✅ 所有核心 JS 库已本地化（lib/ 目录）
```

### 1.2 技术选型理由

| 技术 | 选型理由 |
|------|---------|
| Vue 3（Composition API） | 响应式数据绑定，适合复杂 UI 状态管理 |
| Lightweight Charts v5.0.6 | 轻量级、高性能 K 线图库，支持指标叠加 |
| 原生 JS（非 webpack） | 零构建步骤，直接浏览器打开即可运行 |
| 内嵌数据方案 | 绕过 file:// 协议的 CORS 限制，实现零服务器运行 |
| Python 脚本（生成数据） | 数据处理方便，但非运行时依赖 |

### 2.3 `js/indicators.js` — 技术指标计算

**职责**：
- 提供纯函数计算各类技术指标
- 支持两种输入：数字数组 或 K 线对象数组
- 全局导出到 `window.Indicators`

**导出接口**：
```javascript
window.Indicators = {
  SMA:      calcMA,       // 简单移动平均
  MACD:     calcMACD,     // MACD 指标
  RSI:      calcRSI,      // RSI 指标
  KDJ:      calcKDJ,      // KDJ 指标
  BOLL:     calcBOLL,     // 布林带
  calcAllIndicators        // 一次性计算所有指标
};
```

**算法说明**：
- **MA**：简单移动平均，公式 `sum(window) / period`
- **EMA**：指数移动平均，公式 `EMA = price * k + EMA_prev * (1 - k)`，其中 `k = 2 / (period + 1)`
- **MACD**：`DIF = EMA(fast) - EMA(slow)`，`DEA = EMA(DIF, signal)`，`MACD = (DIF - DEA) * 2`
- **RSI**：`RS = avg_gain / avg_loss`，`RSI = 100 - 100 / (1 + RS)`
- **KDJ**：`RSV = (C - L9) / (H9 - L9) * 100`，`K = 2/3 * K_prev + 1/3 * RSV`，`D = 2/3 * D_prev + 1/3 * K`，`J = 3K - 2D`
- **BOLL**：`MID = MA(close, period)`，`UPPER = MID + multiplier * stdDev`，`LOWER = MID - multiplier * stdDev`

### 2.4 `js/backtest-engine.js` — 前端回测引擎

**职责**：
- 提供 `BacktestEngine` 类，纯 JS 实现回测逻辑
- 内置策略函数（双均线、MACD）
- 计算绩效指标（收益率、夏普比率、最大回撤、胜率）

**核心类：`BacktestEngine`**

```javascript

    this.equityCurve = [];    // 资金曲线
    this.positions = [];     // 持仓记录
  }

  run(klineData, strategyFn, options) {

    // 生成交易信号（buy/sell/hold）
    // 模拟成交（考虑手续费和滑点）
    // 记录每日资金曲线
    return this.calculateMetrics(initialCapital);
  }

  calculateMetrics(initialCapital) {
    // 计算：总收益率、年化收益率、最大回撤、夏普比率、胜率
  }
}
```

**内置策略**：

| 策略名 | 函数名 | 信号逻辑 |
|--------|--------|---------|
| 双均线策略 | `dualMAStrategy(short, long)` | 短 MA 上穿长 MA → 买入；下穿 → 卖出 |
| MACD 策略 | `macdStrategy(fast, slow, signal)` | DIF 上穿 DEA → 买入；下穿 → 卖出 |

**回测参数**：
- `capital`：初始资金（默认 100,000 元）
- `commission`：手续费率（默认 0.03%）
- `slippage`：滑点（默认 0.1%）

**输出指标**：
- `totalReturn`：总收益率（%）
- `annualReturn`：年化收益率（%）
- `maxDrawdown`：最大回撤（%）
- `sharpe`：夏普比率
- `trades`：交易次数
- `winRate`：胜率（%）

## 四、关键技术决策

### 4.1 为什么使用内嵌数据方案？

**问题**：`file://` 协议下，浏览器禁止 JS 通过 `fetch` 或 `XMLHttpRequest` 加载本地文件（CORS 限制）。

**解决方案对比**：

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|---------|
| 启动本地 HTTP 服务器 | 标准方案，无兼容性问题 | 需要用户手动启动服务器，增加使用门槛 | ❌ 不采用 |
| 数据内嵌（当前方案） | 双击即用，零配置 | 数据文件变大（但因为是 JS，浏览器缓存友好） | ✅ 采用 |
| Electron 打包 | 专业桌面应用体验 | 需要打包，增加复杂度 | ❌ 过度设计 |

**实现要点**：
- 数据通过 `window.EMBEDDED_KLINE_DATA` 全局变量访问
- `fetchKline()` 优先检查该变量，有数据则直接返回
- 生成脚本：`gen_embedded_data.py`

### 4.2 为什么将 Vue 3 和 Lightweight Charts 本地化？

**问题**：CDN 加载慢，且 Tailwind CSS CDN 是全量编译器（每次在浏览器里实时编译所有 Tailwind 类），导致页面加载极慢。

**解决方案**：
- 下载 Vue 3 和 Lightweight Charts 为本地文件（放入 `lib/` 目录）
- 移除 Tailwind CSS CDN（项目实际未使用 Tailwind 工具类，所有样式均为手写 CSS）
- 结果：页面加载速度从 10+ 秒提升至 1-2 秒

### 4.3 Lightweight Charts v5.0.6 API 要点

**v5 与 v4 的主要区别**：
- v4：`chart.addSeries(type, options)`，其中 `type` 是数字（2 = 蜡烛图，3 = 直方图）
- v5：`chart.addSeries(SeriesConstructor, options)`，其中 `SeriesConstructor` 是 `LightweightCharts.CandlestickSeries` 等构造函数

**当前项目使用的 API（v5 风格）**：
```javascript
// K 线（蜡烛图）
candleSeries = chart.addSeries(LightweightCharts.CandlestickSeries, {
  upColor: '#f85149',    // 上涨（A 股习惯：红涨）
  downColor: '#3fb950',  // 下跌（A 股习惯：绿跌）
  ...
});

// 成交量（直方图）
volumeSeries = chart.addSeries(LightweightCharts.HistogramSeries, {
  priceFormat: { type: 'volume' },
  priceScaleId: '',
  scaleMargins: { top: 0.8, bottom: 0 },
});

// MA 线（折线图）
ma5Series = chart.addSeries(LightweightCharts.LineSeries, {
  color: '#f6c000',
  lineWidth: 1,
  ...
});
```

**Crosshair（十字线）配置**：
- 当前已禁用默认 crosshair 显示（`vertLine: { visible: false }`），避免首次渲染时的伪影问题
- 如需启用，可在用户鼠标悬停时动态显示

## 六、性能优化要点

| 优化点 | 实现方式 |
|--------|---------|
| JS 库本地化 | 下载 Vue 3 和 LWC 为本地文件，避免 CDN 延迟 |
| 移除 Tailwind CDN | 项目未使用 Tailwind 工具类，移除后无影响但大幅提升加载速度 |
| 内嵌数据优先 | 避免运行时发起网络请求加载数据 |
| `requestAnimationFrame` | 图表渲染放入 rAF 回调，避免阻塞 UI |
| Lightweight Charts 内置优化 | 该库已针对大量数据点优化（800 条数据无压力） |
